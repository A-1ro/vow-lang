## お題(プロンプト本文に書いた自然言語のみ。Kei 語彙ゼロ)

> 在庫を表す小さなデータを複数受け取って、各データには商品 ID(整数)と数量(整数)が入っています。
> 数量が最低発注数(10 個)以上に達している商品だけを残してください。
> 残ったものの数量の合計を返します。
> ただし、残った対象が 1 件も無い場合、もしくは合計が 1000 を超えた場合は失敗として扱います。
> それ以外は成功として合計値を返してください。

## 結果

**成功**(初回 `kei_check` でエラーゼロ)。サブエージェント自己採点: 8/10。

### サブエージェントが書いた .kei

```kei
module inventory.filter

record StockItem {
  productId: Int
  quantity: Int
}

enum StockError {
  NoItems
  TotalExceeded { total: Int }
}

func hasMinQuantity(item: StockItem) -> Bool {
  return item.quantity >= 10
}

func addQuantity(acc: Int, item: StockItem) -> Int {
  return acc + item.quantity
}

func sumFilteredStock(items: List<StockItem>) -> Result<Int, StockError> {
  let filtered = items.filter(hasMinQuantity)
  if filtered.isEmpty() {
    return Err(StockError.NoItems)
  }
  let total = filtered.fold(0, addQuantity)
  if total > 1000 {
    return Err(StockError.TotalExceeded { total })
  }
  return Ok(total)
}
```

## 観測された摩擦(構造化フィードバック原文要約)

1. **MCP `kei_check` のインターフェイスが SKILL.md に未記載** — CLI 形式 `kei check <file> --json` しか書かれていないため、MCP 経由で「ファイルパス?ソーステキスト?」が初見では分からない。今回はツールスキーマで自力解決。
2. **コンビネータに渡す関数の型シグネチャ制約が暗黙** — `items.filter(hasMinQuantity)` のような例はあるが、関数参照の引数・戻り型一致の制約は明文化されていない。類推で書けた。
3. **`fold` のクロス型(アキュムレータ型 ≠ 要素型)制約の明文不足** — SKILL.md §2 の `totalQty` 例と同形なので類推できたが、不安は残った。
4. **取説/ツールに辿り着くまでは迷わなかった** — SKILL.md 冒頭の「書いたら必ず `kei check` に通す」で MCP `kei_check` への到達は自然。

お題設計上、以下の v0.4 新機能は **どれも自然な解では踏まれなかった**:
