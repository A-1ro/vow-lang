## お題(プロンプト本文 — Kei 語彙ゼロ)

> レストランの注文セッションを表すデータがあります。各セッションには「現在の状態」と「現時点の合計金額(整数)」が記録されています。状態は次の 3 つのいずれか: 「受付中(これから注文を追加できる)」「締切(もう追加できないが支払いはこれから)」「支払済(完了)」。
>
> このセッションに対して、「新しい注文の金額」を受け取って、その金額を合計に加算した **新しいセッション** を返す関数を 1 つ書いてください。加算が許されるのは状態が「受付中」のときだけ。それ以外のときや、加算しようとする金額が 0 以下のときは失敗を返してください。

## 結果

**成功**(初回 `kei_check` PASS)。サブエージェント自己採点: 8/10。

### サブエージェントが書いた .kei

```kei
module restaurant.order

enum SessionStatus {
  Accepting
  Closed
  Paid
}

record OrderSession {
  status: SessionStatus
  total: Int
}

enum AddOrderError {
  NotAccepting
  InvalidAmount
}

func addOrder(session: OrderSession, amount: Int) -> Result<OrderSession, AddOrderError> {
  if amount <= 0 {
    return Err(AddOrderError.InvalidAmount)
  }
  let newTotal = session.total + amount
  return match session.status {
    SessionStatus.Accepting => Ok(OrderSession { status: SessionStatus.Accepting, total: newTotal })
    SessionStatus.Closed => Err(AddOrderError.NotAccepting)
    SessionStatus.Paid => Err(AddOrderError.NotAccepting)
  }
}
```

## 観測された摩擦

1. **`None()` vs `E.V` の記法 2 系統が初見で混乱** — 組み込み型(Option/Result)は `None()` のように `()` 付き、ユーザー定義 unit バリアントは `Shape.Dot` のように `E.V` 形式。両方とも SKILL.md に明示はあるが、「同じ unit バリアントの概念で 2 つの記法」が並んで現れたため一瞬立ち止まった。
2. **match の腕の中から関数スコープの `let` 束縛が見えるか明示なし** — SKILL.md は match の「パターン束縛」の話に集中していて、外側の関数スコープ変数(`newTotal`)が腕の式から参照できるかは推測で書いた(結果通った)。
3. **`Ok(record リテラル)` の例が examples に無い** — SKILL.md の `Ok(...)` 例は単一変数や簡単な式のみ。`Ok(OrderSession { status: ..., total: ... })` のように record リテラルを直接ネストするパターンを `withdraw.kei` の `Ok(current - amount)` から類推した。

## 助かった点(原文要約)

- **`&&` 禁止の注記が太字で明示** — 事前に気付き、if を 2 つに分けるアプローチに自然に流れた。
- **`match` が式であること、腕は式のみ** がはっきり書かれている — `let newTotal` を match の外に出す設計を選べた。
- **エラー一覧が「症状 → 原因 → 直し方」順** — 今回は使わなかったが安心感あり。
- **最小テンプレート `counter.kei` が冒頭にある** — module 宣言・func シグネチャ・返し方を 8 行で掴めた。
- **`kei_check` がソーステキストを直接受け取る** — 往復コストがほぼゼロ。

## **examples に同型コード例が無くてゼロから組み立てた部分**(本ラウンドの検証主題)

サブエージェント自身が以下を明示:

- `return match expr { Ok(...), Err(...), Err(...) }` の **Result 分岐パターン** — SKILL.md の match 例は Option/Bool しか返していない
- **状態遷移型 record(record × enum を持つ型)を `Ok(record リテラル)` でラップして返す構造** — `withdraw.kei` の `Ok(current - amount)` から類推

それでも初回 PASS した → **SKILL.md の「原則 + 断片例 + エラーカタログ」構成は同型コピペ不可能なドメインでも組み立て可能なレベルに達している**。
