## 観測された摩擦

### 【重要 / 規模で初めて顕在化】`&&` 不在のとき本体 Bool 式で AND をどう書くか

- `requires` を複数並べる方法は SKILL.md に載っているが、**純粋 Bool 関数本体** で AND が必要な場合(`overlaps`)の例が無い
- サブエージェントは入れ子 if パターンで書いた:
  ```kei
  func overlaps(a: StudySession, b: StudySession) -> Bool {
    if a.startAt < b.endAt {
      if b.startAt < a.endAt {
        return true
      }
      return false
    }
    return false
  }
  ```
- 通ったが推測コストが高い。SKILL.md にこのイディオムを明示するか、`&&` を実装するか
- R1〜R3 では純粋 Bool 関数が出なかったため未検出。**規模(機能の多様性)が増えると確実に踏む**

### 規模が大きくなって取説に書かれていない判断が必要だった部分

1. **複数の値を返す wrapper record イディオム** — Kei にタプル無し。「3 つまとめて返す」要件で `BookingResult` を作る判断は「record の用途」からの推論
2. **エラー型の粒度基準** — `addReservation` と `confirmReservation` でエラー型を分けるか共通化するか(別 enum にした)
3. **`result` 識別子の取り扱い** — 予約語リストに `result` が無いが `ensures` での特殊扱いとの衝突が不安で `bookingResult` に変えた

### 複数モジュール分割について

- 今回は 1 ファイルに収めた(60 行強で分割の動機なし)
- → **M20 import 境界は踏まれず**。「規模が大きい」だけでは M20 を踏ませられない。分割せざるを得ない規模(数百行以上)か、ドメインで明示的に分けるお題が必要
