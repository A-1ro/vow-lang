# KEI-E2007: match is not exhaustive

型検査: `match` 式が、スクルティニの取りうる全バリアントを被覆していない。
Kei の `match` は **全ケース明示**を強制する(ワイルドカード `_` は無い)。
`Option<T>` は `Some` / `None`、`Result<T, E>` は `Ok` / `Err`、enum はすべての
バリアントの腕が要る。

```kei
func bad(daysLeft: Option<Int>) -> Int {
  return match daysLeft {
    Some(x) => x
    // error: missing arm(s) for 'None'
  }
}
```

## 修正

不足しているコンストラクタの腕を追加する。Diagnostic は不足バリアントを
列挙する(`missing arm(s) for 'None'`)。enum を網羅していれば、後でバリアントを
足したときに **その match が必ずコンパイルエラーになる**ので、追従漏れを防げる。

```kei
func ok(daysLeft: Option<Int>) -> Int {
  return match daysLeft {
    Some(x) => x
    None => 0
  }
}
```
