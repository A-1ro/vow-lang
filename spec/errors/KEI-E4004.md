# KEI-E4004: only query observers may be called in contracts

契約検査: `requires` / `ensures` の契約式の中で、`extern query` でない外部関数を呼んだ。
契約は状態を**観測**するものであって作用させるものではないため、契約式から呼べる外部関数は
**純粋観測子(`extern query`)だけ**に限る(spec/kei-spec-v0.2.md §4.3 / M14 / #45)。
副作用の有無に関わらず、観測子として明示宣言されていない `extern` は契約から呼べない。

```kei
import infra.library as Library

extern Library.availableOf(book: String) -> Int   // query ではない

func check(book: String) -> Bool
  requires Library.availableOf(book) > 0
  // error: external function 'Library.availableOf' may only be called in a contract
  //        if declared 'extern query'
{
  return true
}
```

## 修正

その外部関数が純粋な観測子(状態を変えない読み取り)なら `extern query` で宣言する。これにより
契約式から `old()` 付きで参照でき、外部状態の事後条件(「在庫がちょうど 1 減る」など)を直接書ける。

```kei
extern query Library.availableOf(book: String) -> Int

func check(book: String) -> Bool
  requires Library.availableOf(book) > 0
{
  return true
}
```

作用を伴う取得なら、本体でエフェクトとして実行し、結果を引数で契約へ渡す(`KEI-E4001` も参照)。
