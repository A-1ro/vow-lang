# KEI-E4008: missing postcondition (contract suggestion)

構造化修正提案(Agent Repair Protocol / M18 / #24)。`kei check --suggest-contracts` の
**warning**(opt-in)。契約(`ensures`)を持たない純粋関数で、本体が単一の `return <式>` の
ものに、本体から導いた事後条件 `ensures result == <式>` を提案する。診断は通常の `fixes` に
加えて `suggested_contract`(構造化提案)を持つ——エージェントが機械適用 → 再検証できる
(spec/diagnostic-schema.md「SuggestedContract」)。

```kei
func decrementAvailable(available: Int) -> Int
  requires available > 0
  # warning: function 'decrementAvailable' has no postcondition;
  #          an ensures can be derived from its body
  # suggested_contract: ensures result == available - 1
{
  return available - 1
}
```

## 修正

提案された `ensures` を加える。導かれた契約は本体と構築上一致するので check-clean になり、
`kei check --generative` では `generative`(反例ゼロ)まで検証レベルが上がる。

```kei
func decrementAvailable(available: Int) -> Int
  requires available > 0
  ensures result == available - 1
{
  return available - 1
}
```

提案された式が**弱すぎる/強すぎる**なら、人間が承認前に書き換える(提案はあくまで出発点で、
契約の正本は人間がレビューする)。これは opt-in の提案で、既定の `kei check` では出ない。
