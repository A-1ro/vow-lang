# KEI-E2009: invalid match pattern

型検査: `match` のパターンがスクルティニの型に合っていない。次のいずれか。

- スクルティニの型と異なるコンストラクタ族を使った(`Result` を `Some` / `None`
  で開く、`Option` を `Ok` / `Err` で開く)。
- enum のパターンを `Enum.Variant` 形で書いていない(`match` の enum パターンは
  構築形と対称に `Color.Red` のように書く)。
- 存在しないバリアントを参照した。
- ペイロードの束縛形・個数が定義と合わない(`Some(x, y)` のように束縛が多すぎる、
  名前付きフィールドの一部しか束縛していない、など)。

```kei
func bad(outcome: Result<Int, String>) -> Int {
  return match outcome {
    Some(x) => x   // error: 'Some' does not match a Result; expected 'Ok(x)' or 'Err(e)'
    None => 0
  }
}
```

## 修正

スクルティニの型に合ったコンストラクタ・束縛形に書き換える。`Option<T>` は
`Some(x)` / `None`、`Result<T, E>` は `Ok(x)` / `Err(e)`、enum は `Enum.Variant`
(ペイロードは位置 `Enum.V(a, b)` または全フィールド `Enum.V { a, b }`)。
