# KEI-E3004: undeclared external call (strict-extern)

エフェクト検査(strict-extern モード): import した名前空間配下の外部呼び出しに
`extern` 署名が無い。既定の `kei check` ではこの呼び出しは opaque で素通りし、
戻り型もエフェクトも検査されない。`kei check --strict-extern` ではこれを **warning**
として指摘する(v0.3 / M16 / #44 — spec/kei-spec-v0.2.md §2.3)。

未宣言のままだと #22 の穴(外部呼び出しのエフェクト漏れ)を検出できない——
例えば `Time.now()` を呼んでいるのに `uses Clock` を書き忘れても気付けない。

```kei
module app.clock

import infra.time as Time

func recordedAt() -> Int {           // uses Clock を書き忘れても既定では通る
  return Time.now()                  // warning(strict): external call 'Time.now' has no 'extern' declaration
}
```

## 修正

その外部呼び出しに `extern` 署名を宣言し、境界を合意書に載せる。署名を書くと
戻り型・エフェクトが検査下に入り、`uses` 宣言漏れ(KEI-E3001)も検出できるようになる。

```kei
extern Time.now() -> Int uses Clock

func recordedAt() -> Int
  uses Clock
{
  return Time.now()
}
```

strict-extern は **opt-in**。既定の `kei check` は未宣言呼び出しを従来どおり通すため、
このコードは `--strict-extern` を付けたときだけ指摘される。
