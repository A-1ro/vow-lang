# KEI-E4003: contract is always false

契約検査: `requires` / `ensures` の契約式が定数畳み込みで **`false`** に確定した。
処理系がコンパイル時に反証済みで、その契約は実行すれば必ず違反する
(`requires false` の関数は呼べば必ず前提違反、`ensures false` の関数は返れば
必ず事後違反)。実行時アサーションへ落とさず、静的エラーとして報告する
(spec/kei-spec-v0.2.md §3 — 健全な静的検証の最小形 / #35)。

```kei
func neverCallable(count: Int) -> Int
  requires false   // error: contract is always false; this 'requires' can never be satisfied
{
  return count
}

func neverReturns(count: Int) -> Int
  requires 1 > 2   // error: contract is always false; this 'requires' can never be satisfied
{
  return count
}

func neverCallable2(name: String) -> String
  requires "a" == "b"   // error: 文字列定数の比較も畳む(常に偽)
{
  return name
}
```

畳める範囲は変数を含まない定数(Int / Bool / String リテラルの算術・比較・論理式)に限る。
SMT・記号実行による一般の契約不能性検出は対象外(spec/kei-spec-v0.2.md §3.3)。

## 修正

契約を実際に満たし得る条件へ書き換える。プレースホルダとして恒偽を置いていた
場合は意図した述語に直す。「この関数は呼べない」を表したいなら、契約ではなく
型(到達不能なバリアントなど)で表現する。
