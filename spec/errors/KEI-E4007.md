# KEI-E4007: seed input does not fit the target function

契約ベース PBT のシード検証(`kei check --generative`): シードの入力が対象関数に適合しない
(spec/kei-spec-v0.2.md §5.3 / M15 / #26)。次のいずれか:

- シードが対象関数の `requires` を満たさない(**無効なシード**)。
- シードの入力名が関数のパラメータに無い / 型が違う / 供給漏れがある。
- シードが存在しない関数を対象にしている。

```text
seeds for decrementAvailable {
  input { available: 0 }
  # error: seed input (available = 0) does not satisfy requires 'available > 0'
  #        of 'decrementAvailable'
}
```

## 修正

`requires` を満たす入力を与える(上の例なら `available` を正にする)。パラメータ名・型・個数を
関数のシグネチャに合わせる。シードは「対象関数が**受け付ける**入力」だけを供給する——
生成テストのオラクルは契約(`ensures`)が担う(`KEI-E4006` も参照)。
