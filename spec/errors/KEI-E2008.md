# KEI-E2008: unreachable match arm

型検査: 同じコンストラクタの腕が `match` に複数ある。先に書いた腕が常に先に
マッチするため、後の腕は到達不能。冗長な腕は意図のあいまいさを生むので禁止する。

```kei
func bad(daysLeft: Option<Int>) -> Int {
  return match daysLeft {
    Some(x) => x
    Some(y) => 0   // error: 'Some' is already covered above
    None => 0
  }
}
```

## 修正

重複した腕を削除する。条件で枝を分けたい場合は、腕の本体側(または呼び出し前の
`requires` / `if`)で表現する。`match` の腕はコンストラクタごとに 1 つ。
