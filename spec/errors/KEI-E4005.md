# KEI-E4005: ensures violated by a generated input (counterexample)

契約ベース PBT(`kei check --generative`): 契約から生成した property-based test で、ある入力が
`ensures` を破った。報告には**最小化された反例入力**が付く(spec/kei-spec-v0.2.md §5 / M15 / #26)。
反例は「実装が契約に合っていない」ことの証拠で、実装(または契約)の修正を要する。

```kei
func decrementAvailable(available: Int) -> Int
  requires available > 0
  ensures result == old(available) - 1
{
  return available - 2
  // error: ensures 'result == old(available) - 1' of 'decrementAvailable'
  //        is violated by a generated input (available = 1)
}
```

## 修正

実装を契約に合わせる(上の例なら `return available - 1`)。契約の方が誤っているなら契約を直す。
**捏造不能性**(§5.1)により、テストを通す方法は「実装を契約に合わせる」ことだけ——生成器も
シードも期待値を持たないので、期待値を歪めて通すことはできない。
